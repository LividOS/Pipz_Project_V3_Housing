FILE TYPE = .json
MANDATED HEADER STYLE = NONE (JSON FILES MUST REMAIN PURE)

RATIONALE:
- JSON does NOT support comments.
- package.json, tsconfig.json, and similar files are parsed by strict toolchains
  (npm, Node, TypeScript, VS Code extension host).
- ANY header or comment inside a .json file WILL break tooling.

GOVERNANCE REQUIREMENT (MANDATORY):

ALL GOVERNED .json FILES MUST USE A SIDECAR GOVERNANCE FILE.

SIDECAR FILE NAMING RULE:
- <original_filename>.governance.txt

EXAMPLES:
- package.json               → package.json.governance.txt
- tsconfig.json              → tsconfig.json.governance.txt
- settings.schema.json       → settings.schema.json.governance.txt

SIDECAR FILE REQUIREMENTS:
- MUST contain a full MAINTEMPLATE header (standard .txt format)
- MUST include GEMINI MEM TAG with FULL PATH pointing to the SIDECAR FILE
- MUST reference the governed JSON file explicitly in content
- MUST receive Content-Fingerprint updates on save
- MUST be listed in INDEXIDLIST.txt as the authoritative governance artifact

THE .json FILE ITSELF:
- MUST remain valid, comment-free JSON
- MUST NOT contain governance metadata
- IS CONSIDERED A RUNTIME / TOOL-CONSTRAINED ARTIFACT
